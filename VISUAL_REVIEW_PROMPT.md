## PHASE 2: PAGE-BY-PAGE VISUAL AUDIT

For EACH page, review the screenshots and the source code. Write findings to VISUAL_REVIEW_FINDINGS.md. Check:

### Typography & Readability
- Is body text bright enough to read comfortably? (should be white/80 minimum for body, white/90 for important text)
- Are there sections where text is too dim/low-contrast against the dark background?
- Is font size consistent between similar elements across pages?
- Are there places where ALL CAPS is overused or inconsistent?
- Is the monospace vs sans-serif split applied consistently?
- Are there walls of text that need visual breaks?

### Color Usage (CRITICAL — per BRAND_TOOLKIT.md)
- ONLY orange (#ff6b35) and white (at varying opacities) should be used
- Are there ANY remaining green (#00ff88), cyan (#00d4ff), or purple (#c084fc) references?
- Is orange being used too aggressively? (every other word bold orange = hard to read)
- Are there sections where everything is the same color and you can't distinguish hierarchy?
- In charts/graphs: are data series distinguishable? (orange + white + light peach)

### Bold & Emphasis
- Is bold text used sparingly and for genuine emphasis, or is half the page bold?
- Are there sections where bold orange + regular orange + bold white all compete?
- Rule: Bold should draw the eye to ONE thing per section, not everything

### Spacing & Layout
- Is vertical spacing between sections consistent?
- Are there sections with too much or too little whitespace?
- Do cards and tiles have consistent padding?
- Are section dividers consistent in style and spacing?

### Cards & Components
- Do all cards use the same border-l-[3px] pattern or are there mixed styles?
- Are card backgrounds consistent (bg-[#12121a] vs bg-white/[0.02] vs bg-[#0a0a0f])?
- Are hover states consistent?
- Do all tables use the same alternating row pattern?

### Mobile Responsiveness
- Does text overflow or get cut off on mobile?
- Do cards stack properly?
- Are tables readable on mobile (horizontal scroll vs reflow)?
- Is the sidebar hidden/handled properly on mobile?
- Are tap targets large enough?

### Visual Hierarchy
- Can you tell what's most important on each page within 3 seconds?
- Is there a clear reading order?
- Are section boundaries clear?
- Does the page feel like it has rhythm or does it feel monotonous?

### Polish & Details
- Are there any orphaned headers (header at bottom of viewport with content below fold)?
- Are loading states handled?
- Are empty states handled?
- Are there any console errors?
- Do all links work?
- Do all images/videos load?

Write findings as:
```
## [PAGE NAME]
### Issue: [short description]
- **Severity:** CRITICAL / HIGH / MEDIUM / LOW
- **Screenshot:** {page}-{viewport}.png, [area description]
- **Current:** [what it looks like now]
- **Recommended:** [what it should look like]
- **Code location:** src/app/[path]/page.tsx ~L[number]
```
